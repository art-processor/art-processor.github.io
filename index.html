ART - Adaptive Reuse Technology

About ART

ART is a Variability Management Technique (VMT) using Frame Technology (FT).

Variability Management Techniques (VMT)

help developers manage the base code for families of system variants, e.g., a family of Linux versions running on various platforms, or financial systems used in different companies. VMTs organize and/or instrument the base code for ease of adaptation and reuse.

Frame technology (FT)

is a language-neutral (i.e., processes various languages) system that manufactures custom software from reusable, machine-adaptable building blocks, called frames. FT is used to reduce the time, effort, and errors involved in the design, construction, and evolution of large, complex software systems.

ART is here to help you automate common every day software engineering task through code generation and to help you eliminate human coding mistakes and achieve better code quality. However almost all our examples will talk about examples and usage in software engineering, ART has been proven to be useful in other aspects as well. In example for description to image generation (link, desc) and mass document handling (link, desc).

How it Works

ART is a simple text processor, reads text files line by line and outputs text files. If the current line in the currently read file is an ART command (#[cmd]), it will be processed, otherwise it will be emitted to the output.

Adapting input files

The ART Processor starts processing with the start file a.k.a specification (SPC) file. ART commands in the start file, and in each subsequently adapted file, are processed in the sequence in which they appear. Whenever the processor encounters an #adapt command in the currently processed file, processing of the current file is suspended and the Processor starts processing the given file. Once processing of the file is completed, the Processor resumes processing of the current file just after #adapt command. In that way the processing ends when the Processor reaches the end of the start file.

However we have not written source code yet, we can already have an idea of where to use this kind of technology. In the earlier example we can identify a declaration part and a template part. First we set up our values we want to use, then we adapt the template with our variables to get the result we were expecting.

Since the template does not care about our specific settings, it can be used by others, with different values without any changes on the template. I can run this with my own set of values, then my neighbor can run it with his own set of values, we both get the output we wanted, even though we were running the through the same template without any changes to it.
